Two-Package Architecture

Structure and style — cleanly separated.

C

@bloc-ui/core

Structure, behaviour, and accessibility. Barebone components with zero visual opinion — every colour goes through CSS custom properties with neutral-grey fallbacks.

npm i @bloc-ui/core
T

@bloc-ui/theme

Optional design layer. CSS custom properties for colours, typography, border radii, and full dark mode — drop it in and every core component lights up.

npm i @bloc-ui/theme

Cascade-Friendly by Design

All base styles use :where() for zero specificity. Your classes and Tailwind utilities always win — no !important needed.

Highest Priority
Your Classes
Consumer styles always win.
Mid Priority
Theme Tokens

Barebone Styles
